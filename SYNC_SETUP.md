# ふしぎノート クラウド同期 セットアップ手順

Mac と iPad（や他の端末）で **同じ内容を自動同期** するための一度きりの設定です。
所要時間 約10分。費用は無料枠で十分です。

---

## しくみ（かんたんに）

```
   Mac で足す ──┐                     ┌── iPad で凪さんが描く
                ▼                     ▼
        ┌──────────────────────────────────┐
        │  Firebase Realtime Database       │ ← クラウドの共有保管庫
        └──────────────────────────────────┘
                ▲                     ▲
    Mac に反映 ─┘                     └── iPad に反映（数秒）
```

- 片方で足した **写真・テーマ・きょうのふしぎ・アイデア** が、もう片方に数秒で出ます。
- 両方の追加を **マージ**（合体）するので、Mac で仕込み・iPad で描く共創ループがそのまま回ります。
- 凪さんのデータは「**家族の合言葉**」から作った秘密のパスにだけ保存されます。
  合言葉はアプリのコードには書かないので、サイトが公開でも他人は中身を見られません。

---

## ① Firebase プロジェクトを作る

1. https://console.firebase.google.com を開く（Googleアカウントでログイン）
2. **「プロジェクトを追加」** → 名前は `fushigi-note` など → 作成
   （Googleアナリティクスは「無効」でOK）

## ② Realtime Database を有効化

1. 左メニュー **「構築 > Realtime Database」**
2. **「データベースを作成」**
3. ロケーションは任意（例: `asia-southeast1`）
4. セキュリティルールは **「ロックモードで開始」** を選ぶ（あとで③で設定）
5. できたら画面上部の URL（`https://xxxx-default-rtdb.firebaseio.com`）を控える

## ③ ルール（アクセス制限）を貼る

Realtime Database の **「ルール」** タブを開き、以下を貼って **公開**：

```json
{
  "rules": {
    "families": {
      "$family": {
        ".read": "auth != null",
        ".write": "auth != null"
      }
    }
  }
}
```

→ ログイン済みの端末が、合言葉から導いた自分のパスにだけ読み書きできます。
　 一覧で他の家族を覗くことはできません。

## ④ 匿名ログインを有効化

1. 左メニュー **「構築 > Authentication」** → **「始める」**
2. **「Sign-in method」** タブ → **「匿名」** を選んで **有効** → 保存

## ⑤ ウェブアプリを登録して設定をコピー

1. 左上の歯車 **「プロジェクトの設定」**
2. 下の方の **「マイアプリ」** → **`</>`（ウェブ）** アイコンをクリック
3. ニックネーム `fushigi-note` などを入れて登録（Hosting は不要）
4. 表示される `const firebaseConfig = { ... }` の中身をコピー

## ⑥ `firebase-config.js` に貼る

このフォルダの `firebase-config.js` を開き、`PASTE...` の部分を④でコピーした値に置き換える：

```js
window.FUSHIGI_FIREBASE_CONFIG = {
  apiKey: "AIza...",
  authDomain: "fushigi-note.firebaseapp.com",
  databaseURL: "https://fushigi-note-default-rtdb.firebaseio.com",
  projectId: "fushigi-note",
  appId: "1:1234...:web:abcd...",
};
```

※ `databaseURL` は②で控えた URL（無いと同期しません）。

## ⑦ 公開（デプロイ）

変更を GitHub に push して、いつもの手順で公開。
（Service Worker のキャッシュ名は v10 に上げてあるので、開き直すと新版が入ります）

## ⑧ 各端末で「合言葉」を入れる

1. アプリの **おとな設定 → ☁ クラウド同期** を開く
2. 家族の合言葉（例: `nagi-kazoku-2026`）を入力
3. **Mac と iPad で まったく同じ合言葉** を入れる → 数秒で同期開始

合言葉は推測されにくい、少し長めのものにしてください（例に名前＋年など）。
空にして保存すると、その端末だけ同期オフにできます。

---

## 注意・制限（v1）

- **追加は同期、編集・削除は端末ごと**：既存の「バックアップ＝マージ」と同じ挙動です。
  片方で項目を削除しても、もう片方に残っていると次の同期で復活することがあります。
  （写真やアイデアを「足していく」共創ループには最適。一斉削除は各端末で行ってください）
- PIN・メール送り先・「いま使用中のテーマ」は端末ごと（同期しません）。
- オフライン時は今まで通りローカルで動き、オンラインに戻ると自動で同期します。

将来、編集・削除も完全同期したい場合は、項目ごとに更新時刻を持たせる方式（v2）に
拡張できます。必要になったら相談してください。
